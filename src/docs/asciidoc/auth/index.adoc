=== 소셜 로그인 기대 동작 흐름

1. 클라이언트가 OAuth2 인증을 마치고 콜백을 호출하면 서버는 `provider + providerUserId`로 기존 연동 계정을 먼저 조회합니다.
2. 이미 연동된 소셜 계정이 있으면 해당 `member`로 로그인합니다.
3. 연동된 소셜 계정이 없고, 동일한 이메일을 사용하는 기존 회원이 있으면 신규 회원을 만들지 않고 그 `member`에 소셜 계정을 연결합니다.
4. 동일한 이메일 회원도 없으면 신규 소셜 회원을 생성합니다.
5. 신규 소셜 회원 생성 시 닉네임 후보가 기존 회원과 충돌하면 다른 suffix로 재시도합니다.
6. 닉네임 충돌은 서버 내부에서 흡수하며, 단순 unique 충돌 때문에 소셜 가입이 실패하지 않아야 합니다.
7. 하나의 `member`에는 여러 개의 `social_account`가 연결될 수 있습니다.
