== OAuth2 소셜 로그인 API

=== 소셜 로그인 기대 동작 흐름

1. 프론트엔드가 직접 OAuth2 인가 서버(카카오, 구글)로 리다이렉트합니다. (redirect_uri는 프론트엔드 URL)
2. 사용자가 인증을 마치면 인가 서버가 프론트엔드로 인가 코드(`code`)를 전달합니다.
3. 프론트엔드가 받은 인가 코드를 `POST /api/v1/auth/oauth/{provider}` API로 전달합니다.
4. 서버는 `provider + providerUserId`로 기존 연동 계정을 먼저 조회합니다.
5. 이미 연동된 소셜 계정이 있으면 해당 `member`로 로그인합니다.
6. 연동된 소셜 계정이 없고, 동일한 이메일을 사용하는 기존 회원이 있으면 신규 회원을 만들지 않고 그 `member`에 소셜 계정을 연결합니다.
7. 동일한 이메일 회원도 없으면 신규 소셜 회원을 생성합니다.
8. 신규 소셜 회원 생성 시 닉네임 후보가 기존 회원과 충돌하면 다른 suffix로 재시도합니다.
9. 하나의 `member`에는 여러 개의 `social_account`가 연결될 수 있습니다.
