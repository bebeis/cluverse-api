== 그룹 API

=== 그룹 도메인 개요

그룹 도메인은 `그룹 메타데이터`와 `그룹 전용 게시판`으로 나뉜다.

- 그룹은 이름, 소개, 대표 이미지, 카테고리, 활동 방식, 지역, 공개 범위, 멤버, 직책 같은 모임 정보를 담당한다.
- 그룹 전용 게시판은 그룹 내부 공지/일반 게시글을 담당한다.
- 그룹 상세 응답에는 항상 `boardId`가 포함되며, 프론트는 이 값을 사용해 그룹 게시판과 게시글 API를 호출해야 한다.

즉, `groupId`는 그룹 상세/멤버/직책 관리에 사용하고, `boardId`는 그룹 게시판/게시글 조회와 작성에 사용한다.

=== 그룹 상태와 라이프사이클

- 그룹 생성 시 서버는 그룹 전용 게시판을 함께 생성한다.
- 그룹 생성 시 생성자는 자동으로 `OWNER` 멤버가 된다.
- 그룹 생성 시 기본 커스텀 직책 `운영진`, `멤버`가 자동 생성된다.
- 그룹 생성 시 오너에게는 기본적으로 `운영진` 직책이 부여된다.
- 그룹 수정 시 그룹의 `name`, `description` 변경은 연결된 그룹 게시판의 이름/설명에도 함께 반영된다.
- 그룹 삭제 API는 물리 삭제가 아니라 종료 처리이며, 그룹 상태를 `CLOSED`로 바꾸고 연결된 게시판을 비활성화한다.
- 종료된 그룹은 그룹 목록/내 그룹 목록/상세 조회에서 제외된다.

=== 권한 모델

그룹에는 `기본 권한 역할`과 `커스텀 직책`이 따로 존재한다.

- 기본 권한 역할: `OWNER`, `ADMIN`, `MEMBER`
- 커스텀 직책: `운영진`, `디자이너`, `백엔드 리드`처럼 화면 표시용 이름

프론트에서는 다음처럼 구분해서 사용한다.

- `myRole`, `member.role`은 권한 판단용 값이다.
- `roles`, `member.customTitle`은 화면 표시용 값이다.

그룹 게시판의 권한 규칙은 다음과 같다.

- 그룹 게시판 읽기: 그룹 멤버만 가능
- 그룹 게시판 일반 글 작성: 그룹 멤버 가능
- 그룹 공지 작성/관리: `OWNER`, `ADMIN` 가능
- 종료된 그룹 게시판: 기존 멤버도 접근 불가

=== 프론트 연동 흐름

일반적인 화면 흐름은 다음과 같다.

1. 그룹 목록 화면: `GET /api/v1/groups`
2. 내 그룹 목록 화면: `GET /api/v1/groups/me`
3. 그룹 상세 화면: `GET /api/v1/groups/{groupId}`
4. 그룹 게시판 홈 화면: 그룹 상세 응답의 `boardId`로 `GET /api/v1/boards/{boardId}/home`
5. 그룹 게시글 목록 화면: `GET /api/v1/posts?boardId={boardId}`
6. 그룹 게시글 작성: `POST /api/v1/posts` 요청 본문에 `boardId` 포함

프론트에서 특히 주의할 점은 다음과 같다.

- 게시글 API는 `groupId`가 아니라 `boardId` 기준으로 동작한다.
- 그룹 상세 응답의 `boardId`를 저장해 두고 그룹 게시판/게시글 화면으로 연결해야 한다.
- 게시판 홈 응답의 `isReadable`, `isWritable`, `isManageable`, `postingPolicy`, `tabs` 값을 기준으로 버튼과 탭을 노출하면 된다.
